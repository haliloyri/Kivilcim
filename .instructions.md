# GitHub Copilot Instructions

## Response Format

- Do not include a summary section after completing tasks
- Only confirm completion briefly if the user asks for it
- Focus on the implementation, not on summarizing what was done
- Remove lines like "## Yapılan Değişiklikler" or "### Changes Made"

## Communication Style

- Be direct and concise
- Skip unnecessary explanations for successful completions
- Only provide details if there are errors or if the user asks for them
